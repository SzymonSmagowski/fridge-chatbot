# fridge-chatbot / frontend

Next.js frontend for the fridge chatbot — a 4-tab kiosk SPA.

## Stack

- Next.js 16.2.4 (App Router, Turbopack), React 19.2.4, TypeScript 5
- Tailwind CSS 4, shadcn 4.3 (`components/ui/`)
- `@assistant-ui/react` 0.12.25 — UI components only (Thread, Composer, MessageList, etc.)
- Paraglide JS — compile-time i18n, localStorage strategy. Locales: `en`, `pl` (Polish plural rules baked in).
- sonner — toast notifications
- Runtime: custom (`lib/use-fridge-runtime.ts`) — REST + WebSocket to FastAPI backend, **no vendor SDK**

## Key architectural rule

`@assistant-ui/react` is used for its UI primitives only. Do NOT introduce LangGraph Cloud client, Assistant Transport, or any vendor runtime adapter. All backend communication goes through `lib/api.ts` (REST) and the WebSocket in `lib/use-fridge-runtime.ts`.

## i18n pattern

Translations live in `messages/en.json` and `messages/pl.json`. Paraglide generates typed `m.*` functions under `src/paraglide/`. Run `pnpm run paraglide:compile` after editing message files (also runs automatically as part of `dev`, `build`, `test`). Language toggle in Settings persists locale to localStorage.

## Run

```bash
cd apps/fridge-chatbot/frontend
cp .env.example .env.local   # set NEXT_PUBLIC_BACKEND_URL if needed
pnpm install
pnpm dev                     # http://localhost:3000
```

Or use `./apps/fridge-chatbot/dev.sh` from the repo root to start backend + frontend together.

## Wake-word activation

The kiosk listens passively for "Hey Jarvis" and opens the voice overlay automatically when it hears it. Pipeline:

- **Audio capture:** `AudioWorkletNode` at 16 kHz mono → 80 ms (1280-sample) chunks → main thread.
- **Inference:** `onnxruntime-web` (WASM) running three openWakeWord models in sequence — `melspectrogram.onnx` → `embedding_model.onnx` (Google `speech_embedding`) → `hey_jarvis_v0.1.onnx`. Implementation in `src/lib/wake-word-pipeline.ts`.
- **Detection threshold:** 0.5 probability; 2-second cooldown between triggers (suppresses double-fires from a single utterance).
- **Mic arbitration:** while the voice overlay is open, the listener is paused so LiveKit can claim the mic without a fight (`enabled={!voiceOpen}` in `app-shell.tsx`).

**Setup is automatic** — `dev.sh` calls `scripts/download-wake-word-models.sh` to fetch the three ONNX files (~3.7 MB total) from the openWakeWord v0.5.1 GitHub release on first start. The script is idempotent. Models live in `public/wake-words/` (gitignored).

**No API key, no account, no licence, no expiry.** openWakeWord is Apache-2.0; mic audio never leaves the device.

**Why "Hey Jarvis" and not "Hej lodówko":** openWakeWord ships pretrained classifiers for ~10 English phrases. Custom phrases (Polish included) require training a new classifier — synthetic data via Piper TTS + ~30 min on a free Colab GPU. The wake word is just a *trigger*; once the overlay opens, the LangGraph `detect_language` node picks Polish/English per turn from the actual content. Training a "Hej lodówko" classifier is a future single-afternoon project; the wiring already supports a swappable `.onnx` (just change the path + tensor names in `wake-word-pipeline.ts`).

## Adding components

```bash
pnpm dlx shadcn@latest add <component-url>            # shadcn primitive → components/ui/
pnpm dlx assistant-ui@latest add <component>          # owned code → components/assistant-ui/
```

New pages go under `src/app/<route>/page.tsx` (App Router convention).

## Backend contract

Auth via JWT from `POST /auth/login`. REST at `http://localhost:8001`. Chat WebSocket: `ws://localhost:8001/ws/threads/{id}` (first message = `{"token":"<jwt>"}`). Family events WebSocket: `ws://localhost:8001/ws/family/{family_id}/events?token=<jwt>`.
